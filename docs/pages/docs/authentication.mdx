# Authentication

All API requests require an `api_key` query parameter.

## Authentication Model

The API key is issued and managed by the EDU team (not by Immosurance).

Include your key in each request:

```
?api_key=YOUR_EDU_API_KEY
```

**Example:**

```
GET https://edu.immosurance.net/api/immosurance/users/exists?immosurance_user_id=user-123&api_key=YOUR_EDU_API_KEY
```

- Store the API key securely on your side

## Common Errors

| Status | Message | Cause |
|--------|---------|-------|
| 401 | API key is required | Missing `api_key` parameter |
| 401 | Invalid API key | Unknown or invalid key |
| 403 | Forbidden | Caller is not allowed for the requested action |
| 404 | User not found | User does not exist |
| 404 | Organization not found | Organization does not exist |
| 422 | Validation error | Invalid request body or parameters |
