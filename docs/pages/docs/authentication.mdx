# Authentication

All API requests require authentication via the `api_key` query parameter.

## API Key

Enter your API key on the [API Key](/api-key) page to use it in the API Reference Playground. Select the **EDU Immosurance API Key** identity there and your key will be sent automatically with requests.

Include your API key in every request:

```
?api_key=YOUR_API_KEY
```

**Example:**

```
GET https://edu.immosurance.net/api/immosurance/users/exists?immosurance_user_id=user-123&api_key=YOUR_API_KEY
```

- **Do not log or expose** your API key
- Store the API key in environment variables or secure secrets management
- Rotate keys if they may have been compromised

## Common Errors

| Status | Message | Cause |
|--------|---------|-------|
| 401 | API key is required | Missing `api_key` parameter |
| 401 | Invalid API key | Invalid or expired API key |
| 404 | User not found | User does not exist |
| 404 | Organization not found | Organization does not exist |
| 422 | Validation error | Invalid request body or parameters |
