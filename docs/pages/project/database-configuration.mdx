# Database & Configuration

## Database

- Engine: MySQL
- Database name: configured via `.env` (`DB_DATABASE`)

## Core Data Domains

| Domain | Examples |
|-------|----------|
| Identity | users, organizations, organization groups |
| Learning | curricula, trainings, assessments, questions |
| Progress | user progress, attempts, certificates |
| Integration | external IDs and sync mappings |
| Audit | activity and change logs |

## Important Environment Variables

| Variable | Required | Description |
|---------|----------|-------------|
| `APP_NAME` | Yes | Laravel app name |
| `APP_ENV` | Yes | Environment (`production`, `local`, ...) |
| `APP_KEY` | Yes | Laravel encryption key |
| `DB_CONNECTION` | Yes | Usually `mysql` |
| `DB_HOST`, `DB_PORT`, `DB_DATABASE`, `DB_USERNAME`, `DB_PASSWORD` | Yes | MySQL connection |
| `CACHE_STORE` | Recommended | Cache backend |
| `SESSION_DRIVER` | Recommended | Session backend |
| `QUEUE_CONNECTION` | Optional | Queue backend |

## Security Notes

- Keep `.env` out of version control.
- Rotate API credentials when exposed.
- Use HTTPS in all environments that expose integrations.
